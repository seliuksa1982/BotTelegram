# Учебный Telegram-бот на aiogram 3

Простой Telegram-бот на Python. Знает команды `/start`, `/help`, `/about` и повторяет любое присланное текстовое сообщение.

Это **учебный проект** — подходит, чтобы познакомиться с библиотекой [aiogram 3](https://docs.aiogram.dev/) и Telegram Bot API.

---

## Что внутри

| Файл | Зачем |
|---|---|
| `bot.py` | Сам код бота |
| `.env` | Файл с секретами (токен бота). **Не попадает в Git!** |
| `.gitignore` | Список того, что Git должен игнорировать |
| `requirements.txt` | Список Python-библиотек, нужных боту |
| `README.md` | Этот файл |

## Пошаговая инструкция для Windows (с нуля)

### Шаг 1. Установи Python

1. Открой сайт <https://www.python.org/downloads/>
2. Скачай Python версии **3.10 или новее**.
3. Запусти установщик. **Важно:** на первом экране поставь галочку **«Add Python to PATH»**, иначе команды `python` и `pip` не будут работать в терминале.
4. Нажми **Install Now** и дождись окончания установки.
5. Открой **PowerShell** (нажми `Win`, набери `powershell`, открой) и проверь, что Python установлен:
   ```powershell
   python --version
   ```
   Должно вывестись что-то вроде `Python 3.12.3`.

### Шаг 2. Получи токен бота у BotFather

1. Открой Telegram и найди бота **@BotFather**.
2. Отправь ему команду `/newbot`.
3. Придумай имя бота (как будет отображаться).
4. Придумай username бота (должен заканчиваться на `bot`, например `my_first_learning_bot`).
5. BotFather пришлёт сообщение со строкой вида:
   ```
   1234567890:AAEhBPxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   Это и есть твой **токен**. Никому его не показывай — у кого токен, тот и управляет ботом.

### Шаг 3. Скачай проект

Если ты получил проект архивом — распакуй его, например, в `C:\Users\<твоё_имя>\Desktop\BotTelegram`.

Если умеешь работать с Git:
```powershell
git clone <ссылка-на-репозиторий>
```

### Шаг 4. Открой папку проекта в терминале

В PowerShell перейди в папку проекта (замени путь на свой):
```powershell
cd C:\Users\<твоё_имя>\Desktop\BotTelegram
```

### Шаг 5. Создай виртуальное окружение

Виртуальное окружение — это «песочница», куда установятся библиотеки только для этого проекта (а не на весь компьютер).

```powershell
python -m venv venv
```

После выполнения появится папка `venv` внутри проекта.

### Шаг 6. Активируй виртуальное окружение

```powershell
.\venv\Scripts\Activate.ps1
```

После активации в начале строки появится `(venv)` — это значит, что окружение работает.

> **Если PowerShell ругается на политики выполнения скриптов,** один раз выполни команду:
> ```powershell
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```
> и подтверди (`Y`). После этого повтори активацию.

### Шаг 7. Установи зависимости

```powershell
pip install -r requirements.txt
```

Эта команда прочитает файл `requirements.txt` и установит всё нужное (aiogram, python-dotenv).

### Шаг 8. Вставь токен в файл `.env`

1. Открой файл `.env` в любом текстовом редакторе (например, Блокнот).
2. Замени `сюда_вставить_токен` на токен, полученный от BotFather. Должно получиться примерно так:
   ```
   BOT_TOKEN=1234567890:AAEhBPxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
3. Сохрани файл.

> ⚠️ **Никогда не выкладывай файл `.env` в открытый доступ** (на GitHub и т.п.). Он уже добавлен в `.gitignore`, и Git его игнорирует.

### Шаг 9. Запусти бота

```powershell
python bot.py
```

Если всё хорошо, ты увидишь в терминале строку:
```
Бот запущен. Нажмите Ctrl+C для остановки.
```

### Шаг 10. Проверь работу

1. Открой Telegram и найди своего бота по username, который ты дал ему в BotFather.
2. Отправь `/start` — бот поздоровается.
3. Попробуй `/help` и `/about`.
4. Напиши любой текст — бот ответит `Вы написали: ...`.

### Чтобы остановить бота

В терминале, где он запущен, нажми `Ctrl + C`.

## Частые проблемы

**`'python' is not recognized as an internal or external command`**
При установке Python ты не поставил галочку «Add Python to PATH». Переустанови Python, не забыв про эту галочку.

**`ModuleNotFoundError: No module named 'aiogram'`**
Ты не активировал виртуальное окружение (или активировал, но не установил зависимости). Выполни шаги 6 и 7 ещё раз.

**`Не найден BOT_TOKEN...`**
В файле `.env` нет токена или он указан неверно. Проверь шаг 8.

**`TelegramUnauthorizedError: Unauthorized`**
Токен в `.env` неправильный. Проверь, что ты скопировал его полностью, без лишних пробелов.
